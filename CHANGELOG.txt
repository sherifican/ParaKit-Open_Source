  About the version numbers:
    - Source releases now use a plain "<app>" form (e.g. 4.5.4.3). Older entries
      below used an "<app>-<htmlrev>" form where the trailing "-N" tracked the
      companion Practice/Preview HTML editions; that "-N" was dropped from 4.5.4.3
      on (Practice Mode v2 is no longer updated -- a v3 / Kit Studio is coming in
      the ParaKit v5 rebuild).
    - The earlier .exe line uses a plain "vX.Y.Z" form. The compiled .exe
      currently lags the .py source on purpose (see README.txt); the entries
      below it are the historical release notes carried over from the .exe repo
      (https://github.com/sherifican/ParaKit---Releases).
    - This is the curated history. A handful of very old 4.0-4.4 rapid-iteration
      micro-versions only ever appeared in the in-app log and aren't repeated
      here; everything meaningful is included.

  v4.7.8.8   --   2026-07-12
------------------------------------------------------------------------------
  Audio -> MIDI tab redesign + new Song Library.

  Audio -> MIDI:
  - Redesigned the tab into a three-column layout: the detection settings on the
    left; Enhanced Detection, Neural Stem Isolation, Settings Profiles and Output
    are now their own cards in the middle with a larger Log beneath them; and
    Advanced / Debug stays on the right.
  - NEW: a "Song Library" (Your Songs) right on the tab, mirroring the YouTube and
    Stem Splitter libraries. Right-click a song, or click its "Create MIDI"
    button, to load that song's lossless drums split straight into the
    Audio -> MIDI input, ready to convert. Search, sort, album art and full-song
    preview are included.
  - The Stem Splitter model choice (htdemucs / htdemucs_ft) is now remembered
    between sessions, so htdemucs_ft users don't have to re-select it every launch.
  - Layout fixes: the ONNX model-file row (Browse / Check / Download Model), the
    ride-cymbal toggle line, and the Neural Stem Isolation "Download Model" button
    no longer clip in the narrower column.

  v4.7.8.7   --   2026-07-12
------------------------------------------------------------------------------
  Clone Hero export + .rlrr import correctness.

  Clone Hero / .rlrr:
  - Fix (important): china and 13" crash cymbal hits were silently dropped when
    converting Paradiddle charts to Clone Hero, and when importing a .rlrr into
    the MIDI Editor or the Practice view. On a typical library that's a large
    share of charts losing cymbals from the chart with no warning. china, 13"
    crash and splash now map to the Crash lane (china is treated as crash
    everywhere, matching the difficulty reducer).
  - The batch converter no longer writes a broken, section-less Clone Hero chart
    when a slot produces no drum events -- it skips that slot with an error in
    the log instead of installing an unplayable song.

  MIDI Editor:
  - The one-time ".parakit_bak" safety backups (created before an in-place lossy
    save) now appear in the "Load MIDI" file dialog, so you can reopen one to
    recover -- they are ordinary MIDI files under the hood.

------------------------------------------------------------------------------
  v4.7.8.6   --   2026-07-12
------------------------------------------------------------------------------
  A focused correctness pass on difficulty reduction and MIDI timing.

  Difficulty reduction:
  - Fix (important): Easy / Medium / Hard charts made with note-reduction now
    keep notes across the WHOLE song instead of thinning toward the end.
    Previously, on longer songs, kick and snare could drop out of the back
    portion of a reduced chart (it played fine, then lanes went quiet). The
    kept notes are now spread evenly across the full length. Expert charts were
    never reduced and are unchanged.

  MIDI timing:
  - Note times now round to the nearest MIDI tick instead of always rounding
    down. This removes a small drift that used to accumulate when you saved the
    same chart repeatedly in the MIDI Editor (about 8 ms over ~10 saves). The
    same rounding was applied to the Audio -> MIDI writer and the
    Sheet-Music -> MIDI export so all of them agree.
  - Fixed a rare doubled note that could appear when the Advanced "Peak
    Detection Interval" and "Trigger Align" options were used together.

  v4.7.8.5   --   2026-07-12
------------------------------------------------------------------------------
  A large stability + correctness pass across the whole app. The big ones:

  Stem Splitter "Your Songs" library:
  - Fix (important): Delete / Rename could also match a DIFFERENT song whose
    name starts the same (acting on "Song" could catch "Song_Remix"). It now
    matches only the exact song's own file set.
  - Typing in the library search no longer re-scans your drive on every
    keystroke, and the ".ogg files exist" badge is more reliable.

  MIDI Editor:
  - NEW: MIDI Input (BETA) button — record drum hits straight into the editor
    from a connected MIDI kit (the same device the Practice tab uses).
  - Fix (data-loss): loading or clearing over unsaved edits now warns first.
  - Fix (data-loss): saving over an original MIDI that contains notes the
    editor can't display now warns you and writes a one-time backup
    (.parakit_bak) beside the file.
  - Fix: flags now follow their notes when you move them, and undo is reliable.
  - Fix: applying a multi-row tempo map no longer scrambles note timing.
  - Fix: "remove duplicates" now respects the "Selected notes only" scope.
  - Fix: audio is released after Stop/Pause, so you can immediately re-split,
    re-tag, or delete that file (no more "file in use" errors).

  Audio -> MIDI:
  - Fix: converting no longer silently overwrites an existing MIDI — it asks.
  - Fix: Auto-Detect BPM now fits the offset to the timeline the chart
    actually plays on, so foreign MIDIs stop drifting out of sync.

  Single Song Creator:
  - Fix: "Use BPM from MIDI" now reads the song's real starting tempo (it
    could previously pick the wrong tempo off a later track).
  - Fix: a typo in the manual Offset field is now caught instead of silently
    becoming 0.

  Preview / Practice:
  - Fix: pausing shows the correct position, and playback no longer drifts
    out of sync over a long song.
  - Smoother playback on very dense charts.

  Song Tester:
  - Fix: now measures against the song's real starting tempo.

  Under the hood:
  - Preference / settings saves are now atomic and report failures instead of
    silently losing data; library renames surface problems instead of faking
    success.
  - Ride detection disclaimer updated to show improved reliability compared to
    earlier versions.

  v4.6.1   --   2026-07-10
------------------------------------------------------------------------------
  - Manual MIDI Note Manager -- your note choices are now tied to each MIDI
    chart instead of one global setting. A fresh MIDI starts at the defaults,
    and re-loading a MIDI you tuned before restores the choices you saved for
    it, so you never have to re-tune the same chart twice. "Reset to defaults"
    now clears just the loaded chart.
  - The MIDI Editor's left lane strip now shows each lane's current output note
    under its name (e.g. "38 Snare", "49 Cr15\""), so you can see at a glance
    what every lane will export. The Hi-Hat lane shows its note plus
    "& per note" because individual hats can still be Closed / Open / Pedal.
  - The Hi-Hat lane now works like the other lanes in the Note Manager: pick
    its MIDI note (Closed 42 / Open 46 / Pedal 44) and new hats you place use
    it -- while existing hats keep their own individual values, and mixed
    open/closed charts still survive to the exported MIDI.
  - Open hi-hats now look distinct in the MIDI Editor: an Open hi-hat (note 46)
    renders hollow with a thin yellow outline, so you can tell open from closed
    hats at a glance. Closed and Pedal hats render as normal filled notes. In
    the velocity lane, open hats keep their filled cyan bar with a thin yellow
    outline (the velocity bars are never hollow).
  - Velocity lane: notes that land on the same beat in different lanes now sit
    side by side instead of stacking on top of each other, so every bar is
    visible and clickable.
  - Fixed the header sometimes coming up in light mode on launch (until you
    clicked the Dark/Light toggle) -- the saved theme is now fully applied from
    the first paint.
  - Fixed the Note Manager's lane colour swatches rendering as empty boxes
    under the dark theme (they now show each lane's colour).
  - Clone Hero export fixes:
      * Crashes now map to the GREEN cymbal instead of sharing the YELLOW
        (hi-hat) lane, so a simultaneous hi-hat + crash no longer collapses into
        one note. Hi-hat = yellow, ride = blue, crash = green, matching standard
        4-lane Pro Drums.
      * Accent and ghost markers now use the correct per-lane note numbers --
        previously a fixed, out-of-spec value was written, so ghost notes could
        read as accents and accents were silently dropped.
      * When a cymbal and a same-colour tom would land on the same tick (which
        .chart can't represent), ParaKit now keeps the cymbal.
  - Help tab: documented the Manual MIDI Note Manager, the per-chart behaviour,
    the lane-strip readout, and how the Hi-Hat lane works.

  v4.6.0   --   2026-07-10
------------------------------------------------------------------------------
  - NEW: Manual MIDI Note Manager (MIDI Editor, next to the Tempo Map button).
    Pick which MIDI note each drum lane writes on export -- e.g. Electric vs
    Acoustic Snare, Crash 15"/17"/Splash, Ride 17"/20"/Bell, high vs low toms.
    Crash & Ride switch the actual drum piece in Paradiddle; every lane's
    choice sets the note e-kits/DAWs receive from the exported MIDI. Saved
    between sessions, with a reset-to-defaults button.
  - NEW: Per-note hi-hat control. Right-click any hi-hat note in the MIDI
    Editor to set it Closed (42) / Open (46) / Pedal (44) -- works on multi-
    selections too, and mixed open/closed charts survive export. The Note
    Manager also has a one-click "set all hats" bulk action.
  - Clone Hero export fixes:
      * Easy/Medium/Hard charts now export with the CORRECT difficulty section
        (everything used to be written as Expert).
      * Ghost/accent notes now actually write to the .chart when enabled (the
        checkboxes previously did nothing to the direct export).
      * With difficulty reduction on, the .chart now matches the reduced .rlrr
        instead of keeping every note (chart-only exports were also skipping
        reduction entirely).
      * Extended e-kit hi-hat notes (21/22/23/26) are no longer silently
        dropped from Paradiddle exports.
      * The Paradiddle->CH converter now uses ONE BPM for the whole chart so
        mixed-BPM difficulty files can't corrupt earlier sections' timing.
  - Safer in-app updates: "Download update now" is now all-or-nothing -- it
    verifies the update manifest matches the release, downloads and checks
    every file BEFORE replacing anything, and rolls back if a step fails.
    A failed update can no longer leave a half-updated install.
  - MIDI Editor fixes: flags now follow notes through Quantize/Soft-Quantize
    (they used to silently vanish off the note); paste/repeat now select
    exactly the pasted copies (duplicate or overlapping notes could confuse
    the selection and later edits could hit the wrong notes).
  - Sheet Music -> MIDI no longer permanently leaks a temporary .mid file per
    conversion when "Save file" is off; temp files are cleaned up on exit.
  - Stem-splitter model downloads finalize atomically (a failed finalize can
    no longer delete your existing model file).
  - Detection Troubleshooter improvements (Quick Start & FAQ tab):
      * More built-in answers -- a kick de-bunching tip (~55 ms dedup) when your
        log/checkboxes point to doubled kicks, tom detection-sensitivity guidance,
        and a note about the detection cleanup pass.
      * Honest "reality check" guidance: it now tells you when a setting has hit
        its useful limit and the fix is manual (in the MIDI Editor) instead of
        endlessly nudging a value back and forth. Some problems (crash-vs-ride,
        tom separation, single wrong hits) are called out as manual by design.
      * The results box is much taller so long suggestion lists are readable, and
        the "What's New" panel was moved to the left so it stops crowding it.

  v4.5.8.3   --   2026-07-07
------------------------------------------------------------------------------
  - MIDI Editor -- Delete and bulk edits no longer hit the WRONG notes. After
    a drag, insert, or quantize re-sorted the notes, the selection could
    silently end up pointing at different notes than the ones you had
    highlighted, so Delete / bulk-edit could remove the wrong ones. Fixed.
  - MIDI Editor -- reclassifying a note (click-to-pick-a-lane or drag up/down)
    now counts as an unsaved change. Before, those edits didn't trigger the
    save prompt on quit (work silently lost) and Send to Song Creator could
    hand off a stale file missing them.
  - Auto-detect BPM actually detects now. The audio BPM search had quietly
    broken and returned roughly the same ~79.5 BPM for nearly every song.
    Your charts still played in sync (the sync offset was always correct),
    but the BPM written into the song -- the measure lines in Clone Hero and
    the difficulty-reduction beat grid -- was wrong. Detection is rebuilt and
    validated; "Use BPM from MIDI file" (the recommended default) was never
    affected.
  - Song titles with characters Windows doesn't allow in file names no longer
    break exports -- a ":" aborted the conversion and a "/" silently created
    nested folders. Folder and file names are now sanitized; the title shown
    in-game stays exactly as you typed it.
  - Create Multiple Songs (folder batch): the template's "skip" / "rename"
    overwrite setting is honored again -- existing song folders were being
    silently overwritten on re-runs.
  - Sheet Music MIDI export no longer drifts out of time by the end of long
    songs (a tiny per-chord timing error used to accumulate).
  - Extractor tool (rlrr -> MIDI) hardening: the batch tab reliably runs a
    batch after mixed single/batch use, one malformed .rlrr no longer hangs
    the window or aborts the whole batch, a preview crash was fixed, and
    same-named batch outputs get unique names instead of overwriting each
    other.
  - Neural Stem Isolation no longer silently falls back to normal detection
    for file names containing "_(" (e.g. "Song_(Live).flac"), and skipped
    stems are now reported in the log. Also fixed: running the separator no
    longer clobbers another file's saved Audio -> MIDI settings.
  - Temp-folder cleanup: ParaKit now tidies up its temporary detection /
    separation folders, which could quietly pile up 100+ MB per run.
  - Practice minigame: remapped keys that aren't letters (digits, punctuation,
    F-keys, numpad) now work instead of leaving a lane silently unplayable.
  - Misc crash guards: a BPM typed as "0" is rejected with a clear message
    instead of erroring mid-conversion; a corrupt saved detection-threshold
    value is corrected instead of crashing detection; a missing-dependency
    error no longer leaves the progress bar spinning forever; and a
    conversion's settings are locked in when it starts, so flipping toggles
    mid-run can't corrupt the run.

  v4.5.5.1   --   2026-07-01
------------------------------------------------------------------------------
  - Fixed the in-app updater so "Download update now" pulls EVERYTHING a new
    version needs, not just the main app file. Before, the button grabbed only
    'ParaKit v4.0.py' -- so if a release also updated supporting files (the
    detection cleanup code/models, stem-separator code, etc.), you could end up
    with a mismatched install. The updater now reads a small manifest of the
    version's files and downloads any that are missing or changed -- skipping the
    ones you already have (so the big models aren't re-downloaded every time) and
    backing up whatever it replaces. From here forward, updating via the button
    gives you a complete, correct update without re-cloning the whole repo.
  - Because the OLD updater couldn't fix itself (anyone who updated in-app before
    this version may already have some out-of-date supporting files), 4.5.5.1
    shows a one-time notice on first launch explaining it, with a single
    "Update supporting files now" button that repairs your install in place --
    or you can re-download / re-clone from GitHub. The update-available popup
    also carries a short reminder about this for the next couple of versions.
